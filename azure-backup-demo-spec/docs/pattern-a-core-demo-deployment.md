# Pattern A: Core Demo Environment - Spec-Driven Deployment

Document version: 1.0

Deployment package: `azure-backup-demo-spec/`

Target audience: sales engineering, customer engineering, solution architecture, demo presenters

Primary objective: deploy a small, repeatable Azure Backup demo environment that protects one Azure VM and one Azure file share, creates short-retention backup policies, sends diagnostics to Log Analytics, and supports a simple restore demonstration.

## 1. Use Case

Pattern A is the baseline Azure Backup demo environment. Use it when the audience needs to see how Azure Backup protects common Azure workloads without enterprise landing-zone complexity.

The deployment proves these capabilities:

- Azure VM backup with a short-retention daily policy.
- Azure Files backup with sample business data.
- Recovery Services vault diagnostics to Log Analytics.
- Backup job and protected-item visibility.
- Soft delete enabled and explained.
- One restore path demonstrated after a recovery point exists.
- Cleanup path tested after the demo.

## 2. Acceptance Criteria

The Pattern A deployment is complete when all of these statements are true:

- Workload, protection, and operations resource groups exist.
- Windows demo VM or Linux demo VM exists with a data disk.
- Storage account and `demo-share` Azure file share exist.
- Recovery Services vault exists with VM and Azure Files policies.
- VM backup protection is enabled or ready to enable through the run script.
- Azure Files backup protection is enabled or ready to enable through the run script.
- At least one backup job has been triggered or a recovery point has been pre-staged.
- Log Analytics receives vault diagnostics.
- Presenter can show one restore workflow from the Azure portal.
- Cleanup script has been reviewed and tested in a disposable subscription.

## 5. Deployment Prerequisites

Run the deployment from a Windows PowerShell or PowerShell 7 terminal at the repository root.

Required local tools:

- Azure CLI signed in to the target tenant.
- Azure CLI Bicep support.
- PowerShell 7 recommended.
- Permission to create subscription-scope deployments and resource groups.
- Permission to create VMs, storage accounts, vaults, Log Analytics, diagnostics, and role assignments if RBAC IDs are provided.

Required Azure providers:

```powershell
az provider register --namespace Microsoft.RecoveryServices
az provider register --namespace Microsoft.DataProtection
az provider register --namespace Microsoft.Compute
az provider register --namespace Microsoft.Storage
az provider register --namespace Microsoft.Network
az provider register --namespace Microsoft.OperationalInsights
```

Confirm the active subscription before deployment:

```powershell
az login
az account show --output table
az account set --subscription "<subscription-name-or-id>"
```

| Parameter | Pattern A value | Reason |

## 8. Deploy Pattern A to Azure

Run from the repository root.

```powershell
Set-Location azure-backup-demo-spec

$adminPassword = Read-Host -AsSecureString "VM admin password"

./scripts/deploy.ps1 `
  -Location canadacentral `
  -EnvironmentName backup-demo `
  -Owner "<owner-name>" `
  -RegionCode cc `
  -AutoDeleteAfter "<yyyy-mm-dd>" `
  -AdminUsername azureadmin `
  -AdminPassword $adminPassword
```

Expected deployment result:

- `rg-backup-demo-core`
- `rg-backup-demo-protection`
- `rg-backup-demo-ops`
- `rsv-backup-demo-cc`
- `vm-win-backup-demo-01`
- `stbackupdemo<unique>`
- `demo-share`
- `law-backup-demo-cc`
- `pol-vm-daily-demo`
- `pol-files-daily-demo`

## 9. Capture Generated Names

The storage account name is generated. Capture it after deployment.

```powershell
$CoreResourceGroupName = "rg-backup-demo-core"
$ProtectionResourceGroupName = "rg-backup-demo-protection"
$VaultName = "rsv-backup-demo-cc"
$VmName = "vm-win-backup-demo-01"
$FileShareName = "demo-share"

$StorageAccountName = az storage account list `
  --resource-group $CoreResourceGroupName `
  --query "[?starts_with(name, 'stbackupdemo')].name | [0]" `
  -o tsv

if (-not $StorageAccountName) { throw "Storage account not found in $CoreResourceGroupName." }

$StorageAccountName
```

## 11. Enable Protection and Trigger Backup

Enable or verify VM protection, trigger a VM backup, and enable Azure Files backup when a storage account is supplied.

```powershell
./scripts/trigger-backup.ps1 `
  -CoreResourceGroupName $CoreResourceGroupName `
  -ProtectionResourceGroupName $ProtectionResourceGroupName `
  -VaultName $VaultName `
  -VmName $VmName `
  -StorageAccountName $StorageAccountName `
  -FileShareName $FileShareName `
  -RetainUntil (Get-Date).AddDays(7)
```

Backup jobs can take time. For live demos, run this step before the presentation and use the existing recovery point during the walkthrough.

## 12. Validate Backup State

Run the validation script.

```powershell
./scripts/validate-backup.ps1 `
  -ProtectionResourceGroupName $ProtectionResourceGroupName `
  -VaultName $VaultName
```

Also validate in the Azure portal:

1. Open `rsv-backup-demo-cc`.
2. Open Backup items.
3. Confirm `vm-win-backup-demo-01` appears under Azure Virtual Machine backup items.
4. Confirm `demo-share` appears under Azure Storage / Azure Files backup items after protection is enabled.
5. Open Backup jobs and confirm the latest job status.
6. Open Monitoring or Logs and confirm diagnostics are available in `law-backup-demo-cc`.

## 13. Demo Restore Workflow

Use one of these restore paths for Pattern A:

| Restore path | Demo value |
|---|---|
| VM file recovery | Best 10-minute demo. Show file-level recovery from a VM recovery point. |
| Restore disk | Good architecture demo. Restore disks from a VM recovery point and explain attach/replace options. |
| Azure Files item restore | Best business-user story. Delete one seeded file and restore it from the Azure file share recovery point. |

Presenter sequence:

1. Show the protected VM and file share.
2. Show backup policy names and retention.
3. Show backup job history.
4. Show recovery point selection.
5. Demonstrate one restore path.
6. Explain soft delete and why immutable vault is not locked for this disposable demo.
7. Show Log Analytics queries from `queries/`.

## 14. Monitoring Queries

Run the checked-in KQL files from the Log Analytics workspace:

- `queries/backup-jobs.kql`
- `queries/failed-backups.kql`
- `queries/vault-activity.kql`

Quick backup jobs query:

```kusto
AzureDiagnostics
| where Category has "AzureBackup"
| project TimeGenerated, Resource, Category, OperationName, ResultType, ResultDescription
| order by TimeGenerated desc
```

## 15. Cleanup

Review soft-delete and immutability implications before cleanup. Pattern A should not lock immutable vault settings.

Interactive cleanup:

```powershell
./scripts/cleanup.ps1
```

Non-interactive cleanup for disposable demo environments:

```powershell
./scripts/cleanup.ps1 -Force
```

Confirm resource group deletion:

```powershell
az group exists --name rg-backup-demo-core
az group exists --name rg-backup-demo-protection
az group exists --name rg-backup-demo-ops
```

Each command should eventually return `false`.

## 16. Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Provider type not found | Resource provider not registered | Run the provider registration commands and retry. |
| VM backup item not found | Protection enablement is still propagating | Wait several minutes and rerun `trigger-backup.ps1`. |
| Azure Files item is missing | Storage account name was not passed to `trigger-backup.ps1` | Capture `$StorageAccountName` and rerun with `-StorageAccountName`. |
| Cleanup fails | Vault has active protected items or soft-delete state | Stop protection/delete backup data only when safe, then retry cleanup. |
| Demo takes too long | First backup job is still running | Trigger backup before the presentation and use a pre-staged recovery point. |

## 17. Pattern A Definition of Done

Pattern A is ready for use when:

- Deployment script completes successfully.
- Demo data is seeded into `demo-share`.
- VM backup job has been triggered.
- Azure Files protection has been enabled or demonstrated.
- Recovery Services vault shows protected items and backup jobs.
- Log Analytics queries return backup diagnostics or activity.
- Presenter can complete one restore workflow.
- Cleanup has been validated or scheduled using the `AutoDeleteAfter` value.
