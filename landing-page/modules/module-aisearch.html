Azure AI Search

Enterprise-scale AI-powered search platform with semantic understanding and hybrid search capabilities

Overview

Azure AI Search (formerly Cognitive Search) provides AI-powered search capabilities for your applications. It combines traditional full-text search with AI-powered understanding, semantic ranking, and vector search for semantic similarity matching.

Built on Apache Lucene, Azure AI Search enables you to build intelligent search experiences that go beyond keyword matching, incorporating natural language understanding and machine learning.

Key Capabilities

  • Hybrid Search: Full-text search combined with semantic ranking for better relevance
  • Vector Search: Semantic similarity search using embeddings from Azure OpenAI
  • AI Enrichment: Built-in skills for NLP, OCR, and entity recognition
  • Semantic Ranking: AI-powered ranking using transformer models
  • Faceted Navigation: Filter and drill-down capabilities for complex queries
  • Multi-language Support: Search across 50+ languages
  • Custom Analyzers: Full control over tokenization and text processing
  • Scalability: Handle massive indexes and concurrent queries

FAQ

What's the difference between full-text and semantic search?

Full-text search matches keywords, while semantic search understands meaning and intent. Semantic ranking uses AI to reorder results based on relevance to the actual meaning of the query.

How do vector searches work?

Vector search uses embeddings (numerical representations) created from text using models like OpenAI's embeddings. Documents with similar meanings have similar vectors, enabling semantic similarity search.

What's the maximum index size?

The limit depends on your tier. Standard tier supports up to 50 GB per service. You can scale up with multiple replicas and partitions for larger deployments.

Can I use it with custom embeddings?

Yes. You can use embeddings from Azure OpenAI, Hugging Face, or your own model and index them directly in Azure AI Search.
