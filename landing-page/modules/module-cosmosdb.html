Azure Cosmos DB

Globally distributed, multi-model database with guaranteed low latency and high availability

Overview

Azure Cosmos DB is a fully managed, globally distributed NoSQL database service designed for modern applications that require global scale with guaranteed sub-10ms latencies at the 99th percentile.

It supports multiple data models including document, key-value, graph, and wide-column, enabling you to choose the right model for your application needs while maintaining consistency across the globe.

Key Capabilities

  • Global Distribution: Replicate data across any Azure region with transparent failover
  • Multi-Model Support: Document (SQL/Core), MongoDB, Cassandra, Gremlin (graph), Table APIs
  • Low Latency: Single-digit millisecond response times worldwide
  • Consistency Models: Choose between strong, bounded staleness, session, or eventual consistency
  • Automatic Indexing: Every property is indexed automatically for fast queries
  • Backup & Restore: Automated continuous backups with point-in-time restore
  • Server-side Scripting: Execute stored procedures, triggers, and UDFs in SQL
  • Real-time Analytics: Integrated Azure Synapse Link for OLAP queries

Pricing Tiers

Provisioned Throughput

Cost: Based on RU/s (Request Units per second) and storage

Pay for reserved throughput. Scaling requires manual adjustment. Best for predictable workloads.

Serverless

Cost: Per-request pricing (~$0.25 per million RUs)

Pay only for consumed requests. Auto-scales, best for variable workloads.

Autoscale

Cost: 10x hourly RU allocation * hourly rate

Automatically scale throughput between 4,000-40,000 RU/s (or higher).

Examples

1,000 RU/s provisioned: ~$50/month | 100 GB storage: ~$25/month

FAQ

What are Request Units (RUs)?

RUs are a unit of throughput that abstracts the cost of database operations. A simple read of a 1-KB document costs 1 RU. Complex queries can cost more depending on complexity and operation size.

How is data replicated?

Data is automatically replicated across all regions you specify with transparent failover. Changes are asynchronously replicated based on your consistency model.

Can I have multi-region writes?

Yes! Enable multi-master across multiple regions. Each region can accept writes, and conflicts are resolved automatically based on your conflict resolution policy.

What's the maximum document size?

Documents can be up to 2 MB in size. Larger data should be split into smaller documents.
