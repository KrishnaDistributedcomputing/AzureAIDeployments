Azure Functions

Serverless compute for event-driven workloads with flexible pricing models

Overview

Azure Functions is a serverless compute service that lets you run code on-demand without managing infrastructure. Execute code in response to events from Azure services, on a schedule, or directly from the internet.

Scale automatically based on demand and pay only for the compute time you consume.

Key Capabilities

  • Multi-language Support: C#, Python, JavaScript, Java, PowerShell, and more
  • Event-driven Execution: Triggers from Azure services, HTTP, or schedules
  • Auto-scaling: Scale from 0 to thousands based on load
  • Flexible Hosting: Consumption, Premium, or Dedicated App Service plans
  • Durable Functions: Orchestrate long-running, stateful workflows
  • Input/Output Bindings: Declaratively connect to data sources and outputs
  • Monitoring & Diagnostics: Built-in Application Insights integration
  • Development Tools: VS Code extensions and local runtime for testing

Use Cases

📧 Event Processing

Process messages from Service Bus, Event Grid, or Event Hubs without servers.

⏰ Scheduled Tasks

Run batch jobs, cleanups, or maintenance on a schedule without managing VMs.

🔌 API Endpoints

Create lightweight HTTP APIs that scale automatically.

🔄 Workflow Orchestration

Use Durable Functions to orchestrate complex, long-running workflows.

📁 File Processing

Automatically process files in Storage when uploaded or modified.

Hosting Plans

Consumption Plan

Cost: $0.20 per 1M executions + execution time

Best for unpredictable, event-driven workloads with cost efficiency.

Premium Plan

Cost: $0.04/hour per vCore + 0.2GB memory

Predictable workloads with better performance and always-warm instances.

Flex Consumption

Cost: $0.000278/vCore-second when running

New pay-per-second model with advanced scaling.

App Service Plan

Cost: Free tier or standard App Service pricing

Fixed cost, for guaranteed capacity or existing plans.

FAQ

What's the execution timeout?

Consumption: 5 minutes, Premium: 30 minutes configurable, Durable Functions can run indefinitely.

Can I use local development?

Yes. Azure Functions Core Tools allow local development and testing before deployment.

What's the cold start issue?

Consumption plan has cold starts. Premium plan has warm instances. Flex Consumption minimizes cold starts.

Can I share code between functions?

Yes. Use shared code in separate library files or create shared bindings.
