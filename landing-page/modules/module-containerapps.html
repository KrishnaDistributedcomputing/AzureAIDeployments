Azure Container Apps

Serverless container platform for running containerized applications at scale

Overview

Azure Container Apps is a serverless container platform that makes it easy to build and deploy containerized applications. It abstracts away the infrastructure complexity while providing the power of containers.

Built on Kubernetes but without the operational overhead, Container Apps provides auto-scaling, event-driven execution, and seamless integration with Azure services.

Key Capabilities

  • Serverless Containers: Run containers without managing Kubernetes
  • Auto-scaling: Scale based on HTTP traffic, message queues, or custom metrics
  • Microservices: Built-in service discovery and inter-container networking
  • Event-driven: Trigger containers from Azure services (Storage, Service Bus, etc.)
  • Managed Certificates: HTTPS automatically with free certificates
  • Blue/Green Deployments: Zero-downtime updates with traffic splitting
  • Dapr Integration: Distributed application runtime for state and pub/sub
  • Revision Management: Multiple versions running simultaneously

Use Cases

🚀 Microservices

Deploy microservices with automatic discovery and load balancing, ideal for polyglot architectures.

🔄 Event-Driven Apps

FAQ

How is Container Apps different from AKS?

Container Apps abstracts Kubernetes away. AKS gives you full Kubernetes control. Container Apps is simpler and cost-effective for microservices.

Can I run stateful applications?

Yes, with Dapr state management. Container Apps also supports persistent volumes for data.

What image registries are supported?

Docker Hub, ACR, and any private registry with provided credentials.

How do I get logs from my containers?

Logs stream to Azure Monitor Log Analytics automatically. Query with KQL.
