Azure Key Vault

Centralized secrets and key management for applications and infrastructure

Overview

Azure Key Vault is a cloud service that safeguards cryptographic keys and secrets used by cloud applications. It helps solve key management challenges and keeps your security credentials safe by centralizing them in a secure, dedicated vault.

With Key Vault, you can encrypt keys and secrets, manage certificates, and control access through Azure's identity platform.

Key Capabilities

  • Secrets Management: Store and manage passwords, connection strings, and API keys
  • Key Management: Create and manage cryptographic keys for data encryption
  • Certificate Management: Manage public/private certificates for SSL/TLS
  • Hardware Security Module (HSM): Premium tier with FIPS 140-2 validated HSMs
  • Role-Based Access Control: Fine-grained access with Azure AD integration
  • Soft Delete & Purge Protection: Recover accidentally deleted vaults and secrets
  • Monitoring & Auditing: Track all access and changes via Azure Monitor
  • Key Rotation: Automate secret rotation and versioning

Use Cases

🔑 Application Secrets

Store application connection strings, API keys, and passwords securely without embedding them in code.

🔒 Database Credentials

Manage database connection credentials centrally with rotation and audit trails.

📜 Certificate Management

Manage SSL/TLS certificates with automated renewal and deployment to Azure services.

🔐 Encryption Keys

Store and manage keys for customer-managed encryption in storage, databases, and services.

🏢 Compliance & Governance

Meet regulatory requirements with comprehensive audit logs and access controls.

FAQ

What's the difference between Standard and Premium?

Standard uses software-backed keys, while Premium (HSM-backed) uses FIPS 140-2 validated hardware modules. Premium is required for highly sensitive workloads.

How do I access Key Vault secrets in applications?

Use managed identities with Azure SDKs, Azure Key Vault references in App Configuration, or Key Vault secret mounts in Kubernetes.

What about secret rotation?

Azure provides flexible rotation policies. You can set custom rotation policies and use Event Grid to trigger updates when secrets rotate.

Can I restrict network access?

Yes. Use Private Link for network isolation, or IP-based firewall rules to restrict access to specific networks.
