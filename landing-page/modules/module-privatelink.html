Azure Private Link

Access Azure services privately over your own virtual networks

Overview

Azure Private Link provides private connectivity to Azure services and customer-hosted services. Access services without exposing traffic to the internet.

Services are accessed through private endpoints that have instances within your virtual networks.

Key Capabilities

  • Private Endpoints: Dedicated network interfaces in your VNet
  • Private DNS Integration: Automatic DNS resolution for private IPs
  • No Internet Exposure: Services not exposed to public internet
  • Same VNet Access: Access services using private IPs
  • Cross-region Support: Connect across regions privately
  • DDoS Protection: Protected from internet-based DDoS attacks
  • Data Exfiltration Prevention: Control what data leaves your VNet
  • Compliance Ready: Enables PCI-DSS, HIPAA, and similar compliance

FAQ

How does DNS work with Private Link?

Services resolve to private IPs when accessed from within the VNet. Public DNS resolves to public IPs outside.

Can I access from on-premises?

Yes. Connect your on-premises network to Azure via VPN or ExpressRoute, then access via Private Link.

Is there a limit on private endpoints?

Yes. Default limit is 1000 per subscription, but can be increased via request.

What services support Private Link?

Most Azure services do: Storage, Databases, OpenAI, Cognitive Services, etc.
