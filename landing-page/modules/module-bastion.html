Azure Bastion

Secure remote access to virtual machines without exposing RDP/SSH ports

Overview

Azure Bastion provides secure and seamless RDP/SSH access to your virtual machines through SSL/TLS without exposing public IP addresses. It's a fully managed PaaS service that harddens access and reduces exposure to brute force attacks.

Connect directly from the Azure Portal using HTML5, eliminating the need for VPN clients or public IPs on VMs.

Key Capabilities

  • No Public IP Required: VMs don't need public IP addresses
  • HTML5 Browser Access: Connect via Azure Portal without additional clients
  • RDP/SSH Support: Connect to both Windows and Linux VMs
  • SSL/TLS Encryption: All connections encrypted over HTTPS
  • Multiple VM Support: Connect to multiple VMs simultaneously
  • Session Recording: Optional NSG rule logging for auditing
  • Automatic Patching: Service maintained by Microsoft
  • RBAC Support: Fine-grained access control via Azure RBAC

FAQ

Do I need to disable NSG for Bastion?

No. Bastion works alongside NSG rules. You can restrict RDP/SSH without Bastion access.

Can Bastion access VMs in peered VNets?

Yes, with VNet peering. You can access VMs in peered networks from Bastion.

What are the bandwidth limits?

Standard: 100 Mbps, Premium: 200 Mbps. Sufficient for typical remote access.

How to audit Bastion connections?

Enable NSG flow logs and resource diagnostics for audit trails.
