Azure Monitor

Comprehensive monitoring, diagnostics, and analytics for applications and infrastructure

Overview

Azure Monitor is Azure's comprehensive solution for collecting, analyzing, and acting on telemetry from cloud and on-premises environments. It helps you maximize availability and performance of your applications and services.

Azure Monitor collects metrics, logs, and traces (observability triad) for complete visibility into your systems.

Key Capabilities

  • Metrics: Numerical values on a time series for performance monitoring
  • Logs: Event data for diagnostics and troubleshooting with KQL queries
  • Application Insights: End-to-end monitoring for web applications and services
  • Alerts: Proactive notifications based on condition thresholds
  • Dashboards: Customizable visualizations of key metrics
  • Workbooks: Interactive reports combining metrics, logs, and visualizations
  • Log Analytics: Kusto Query Language (KQL) for deep analysis
  • VM Insights: Performance and health monitoring for virtual machines

Use Cases

Pricing Structure

Log Analytics (Per GB)

Application Insights

Cost: First 5GB free, then $2.99/GB

Included in Log Analytics pricing.

Metrics & Alerts

Cost: First 10,000 time series free, $0.30 per additional 1,000

Based on number of metric time series.

FAQ

What's KQL and why should I use it?

Kusto Query Language (KQL) is a powerful query language for analyzing logs. It enables complex queries for troubleshooting and analytics.

How long is data retained?

Default retention is 30 days for Log Analytics, up to 730 days available. Retention cost is lower than ingestion.

Can I monitor on-premises resources?

Yes. Use Log Analytics agent or Azure Monitor agents on on-premises machines to send data to Azure.

How do I set up alerting?

Create metric alerts, log alerts, or smart detection rules. Configure action groups for notifications.
