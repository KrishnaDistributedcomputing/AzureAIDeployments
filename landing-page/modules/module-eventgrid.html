Azure Event Grid

Fully managed event routing and distribution platform

Overview

Azure Event Grid is a fully managed event routing service that connects applications and services through event-driven architecture. It enables real-time event distribution at massive scale with simple HTTP-based events.

Event Grid automatically scales to handle any volume of events and provides reliable delivery guarantees.

Key Capabilities

  • Event Routing: Route events from any source to any destination
  • Event Topics: Custom or system topics for event publishing
  • Subscriptions: Filter and route events to handlers
  • Multiple Destinations: Webhook, Service Bus, Queue, Function, Logic App
  • Dead-Letter Support: Capture undeliverable events
  • Retry Policies: Configurable retry and expiration
  • Event Filtering: Advanced filtering on event properties
  • Metrics & Monitoring: Built-in Azure Monitor integration

FAQ

What's the difference between Event Grid and Event Hubs?

Event Grid routes events to handlers, Event Hubs captures and streams data. Use Grid for pub-sub, Hubs for streaming.

How do I guarantee delivery?

Event Grid retries for 24 hours by default, with exponential backoff. Use dead-letter for persistent failures.

Can I use Event Grid with my own applications?

Yes. Publish custom events to custom topics using REST API or SDKs.

What formats are supported?

CloudEvents and Event Grid schema formats are both supported.
