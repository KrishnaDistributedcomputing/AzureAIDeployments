Azure Container Registry

Managed container registry for storing and managing Docker container images

Overview

Azure Container Registry (ACR) is a managed private Docker registry service built on Docker Container Registry. It stores and manages container images and related artifacts securely in Azure.

ACR provides integrated container image scanning, webhooks for CI/CD automation, and multi-region replication capabilities.

Key Capabilities

  • Private Registry: Secure container image storage with access control
  • Multi-architecture Support: Store images for Linux and Windows
  • Geo-replication: Replicate images across regions automatically
  • Container Scanning: Built-in vulnerability scanning with Trivy
  • Webhooks: Trigger CI/CD pipelines on image push/delete
  • Image Retention: Auto-delete old or untagged images
  • Artifact Support: Store Helm charts, OCI artifacts beyond images
  • Task Automation: Build images in Azure without Docker daemon

FAQ

How do I push images to ACR?

Use 'docker push' after logging in with 'az acr login'. Images are authenticated and stored in your private registry.

Can I use ACR with AKS?

Yes. AKS has built-in integration with ACR. Just assign the AKS cluster identity ACRPull role.

What's image retention policy?

Auto-delete untagged images or images older than N days. Useful for cleaning up old builds.

How do I enable vulnerability scanning?

Premium tier includes scanning out of the box. Standard and Basic tiers can enable it as an add-on.
