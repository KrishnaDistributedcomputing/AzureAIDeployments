Azure Virtual Desktop

Deliver secure, scalable Windows desktops and remote apps from Azure — with Entra ID integration, FSLogix profile containers, autoscale, and full session monitoring built in.

View Spec Learn More

Overview

Azure Virtual Desktop (AVD) is a cloud-based desktop and application virtualization service that runs on Azure. It enables IT teams to deliver full Windows 11 multi-session desktops, individual app streams (RemoteApp), and persistent personal desktops — all secured by Microsoft Entra ID and managed through the Azure portal.

The spec provisions the three core AVD control-plane resources: a Host Pool (compute session scheduler), an Application Group (desktop or RemoteApp publisher), and a Workspace (user-facing connection hub) — wired to a Log Analytics workspace for full session and connectivity diagnostics.

Key Capabilities

  • Pooled & Personal Host Pools: Pooled multi-session for cost-efficient shared desktops; Personal for dedicated persistent desktops
  • RemoteApp Publishing: Publish individual applications as streamed windows without delivering the full desktop
  • Autoscale Start/Stop: Automatically start session hosts on demand and shut down idle VMs to reduce compute costs
  • FSLogix Profile Containers: Roaming user profiles stored in Azure Files or Azure NetApp Files for consistent experience across session hosts
  • Entra ID + Conditional Access: MFA, device compliance, and location-based policies applied at sign-in
  • Scheduled Agent Updates: Host agent patched on configurable maintenance windows (Sunday 02:00 in spec default)
  • Session Host Hardening: Integrate with Microsoft Defender for Cloud and Intune for endpoint protection and compliance
  • Full Observability: Session, connection, and host diagnostics streamed to Log Analytics; pre-built Azure Monitor workbooks available

Use Cases

Secure Contractor Access

Provide third-party contractors a managed Windows environment without granting access to corporate devices or on-premises networks.

Call Centre Desktops

Cost-efficient pooled multi-session desktops for high-volume shift workers — thin clients at the endpoint, Azure compute in the cloud.

Regulated Workloads

HIPAA, PCI-DSS, and government-classified workloads benefit from fully auditable sessions, Conditional Access enforcement, and data-never-leaving-Azure patterns.

Legacy App Virtualization

Run legacy Windows apps that can't be modernised via RemoteApp publishing — users get a seamless application window without managing old OS environments locally.

Dependency Wiring
Host Pool

Core session scheduling resource. Configured as Pooled / BreadthFirst with maxSessionLimit: 10 and startVMOnConnect: true. Agent patched on Sunday at 02:00.

Name: vdpool-{environment}-{hash}

Application Group

Desktop Application Group linked to the Host Pool. Publishes the full Windows desktop to assigned Entra ID users or groups. Set applicationGroupType: Desktop.

Name: vdag-desktop-{environment}

Log Analytics

Log Analytics workspace for AVD diagnostic settings. Captures host pool, connection, session host, and agent health events. Retention: 30 days.

SKU: PerGB2018

Acceptance Gates
  • Host Pool, Application Group, and Workspace all provisioned with correct naming convention and tags
  • At least one session host VM registered to the Host Pool (validated via az desktopvirtualization sessionhost list)
  • Entra ID user or group assigned to the Application Group and able to sign in via Windows App or Remote Desktop Client
  • FSLogix profile mounts successfully on sign-in; profile container visible in Azure Files share
  • Session and connectivity diagnostics flowing to Log Analytics within 5 minutes of first sign-in
Deployment Workflow
  1. Deploy AVD control plane: Host Pool + Application Group + Workspace via azure-avd.bicep
  2. Capture registrationToken output from Host Pool (valid 24 hours)
  3. Create session host VMs (Windows 11 Multi-Session + FSLogix gallery image) and register using the token
  4. Configure FSLogix profile container settings on session hosts pointing to Azure Files share
  5. Assign Entra ID users or groups to the Application Group via Azure RBAC (Desktop Virtualization User role)
  6. Configure Conditional Access policy in Entra ID targeting the Windows Virtual Desktop app
  7. Enable autoscale (Start/Stop) on the Host Pool with peak/off-peak schedule
  8. Verify outputs and run acceptance gates: sign-in test, FSLogix mount, diagnostics

How to Deploy

Prerequisites
  • Azure CLI 2.57+ with Bicep extension
  • Resource Group in target region
  • Entra ID P1 or P2 licence for Conditional Access
  • Desktop Virtualization Contributor role
  • Azure Files share for FSLogix profile containers
Deployment Steps
  1. Clone repo and navigate to landing-page/specs/
  2. Set hostPoolType, maxSessionLimit, and loadBalancerType
  3. Run az deployment group create (see below)
  4. Capture registrationToken and register session host VMs
  5. Assign Entra ID users and verify sign-in + FSLogix mount
Azure CLI Example
# Deploy AVD control plane
az deployment group create \
  --resource-group rg-avd-prod \
  --template-file azure-avd.bicep \
  --parameters environment=prod hostPoolType=Pooled maxSessionLimit=10 \
  --query "properties.outputs" -o json

# Assign user to Application Group
az role assignment create \
  --assignee user@contoso.com \
  --role "Desktop Virtualization User" \
  --scope <applicationGroupId>

FAQ

What is the difference between Pooled and Personal host pools?

Pooled host pools share session hosts across multiple users simultaneously (multi-session Windows) — ideal for call centres, contractors, and task workers. Personal host pools assign one dedicated VM per user — best for developers, power users, or workloads requiring persistent local state.

How does autoscale work to control costs?

AVD Autoscale uses a configurable peak/off-peak schedule and session thresholds to start additional session hosts as demand rises and deallocate (not delete) VMs when sessions drop. Combined with startVMOnConnect: true, users trigger VM starts on demand, reducing always-on idle spend.

What is FSLogix and why is it needed?

FSLogix Profile Containers redirect the Windows user profile into a VHD/VHDX file stored on Azure Files or Azure NetApp Files. When a user connects to any session host in the pool, their profile mounts instantly — preserving browser history, app settings, and personal data regardless of which VM handles the session.

How do I register session host VMs to the Host Pool?

The Bicep spec outputs a registrationToken valid for 24 hours. Install the AVD agent on each session host VM and provide the token during installation. Alternatively, use the Azure Image Builder pipeline or Deployment Script to automate session host creation and registration as part of the same deployment.

Can I use Private Link with Azure Virtual Desktop?

Yes. AVD supports Private Link for the control plane (gateway, broker, web access, diagnostics) to prevent traffic from traversing the public internet. Deploy private endpoints for the Host Pool and Workspace resources, and configure Private DNS zones for privatelink.wvd.microsoft.com.
