Frequently Asked Questions

Do we need special tooling to use SDD?
No. Start with a Markdown spec, issue tracker IDs, and your existing test pipeline. Spec-Driven Development is a methodology, not a tool dependency.
How detailed should one spec be?
Keep it feature-sized. One spec should usually map to one pull request or one sprint deliverable. Avoid creating specs that span multiple teams or months of work.
What is the most common beginner mistake?
Using subjective language in acceptance criteria without measurable pass/fail thresholds. Replace words like "should," "nice," and "feels" with concrete metrics: time, error rates, throughput, and field-level behaviors.
