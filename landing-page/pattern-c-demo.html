Pattern C: Modern Workloads

Vault Differentiation, Cloud-Native Architectures & Workload Placement

Cloud-Native First Pattern

Pattern Overview

Pattern C demonstrates how to select the right vault type for different workload families. Recovery Services Vault (RSV) for traditional IaaS, Backup Vault for cloud-native.

🎯 Target Audience Platform teams, DevOps engineers, cloud architects, CCoE (Cloud Center of Excellence)
⏱️ Deployment Time 30-35 minutes (varies based on optional workloads)

Recovery Services Vault (RSV)

Best For: Traditional IaaS workloads

  • Azure VMs (Windows/Linux)
  • Azure Files (SMB/NFS)
  • SQL Server in VM
  • SAP HANA
  • On-Premises Servers

Retention: 1-9999 days
Redundancy: LRS, GRS, RA-GRS
Restore: File, Disk, Full VM

Backup Vault (New)

Best For: Cloud-native workloads

  • Managed Disks (Premium/Standard)
  • Azure Blob (Operational backup)
  • AKS Clusters
  • App Services (preview)
  • Database backups

Retention: SLA-driven (1-35 days)
Redundancy: LRS (immutable)
Restore: Granular snapshots

Workload Placement Matrix

Use this matrix to decide which vault to use for each workload type:

Workload Vault Type Why Restore Strategy
Azure VM RSV Flexible restore options (file/disk/full VM) File-level, Disk, Full VM
Azure Files RSV Snapshot-based point-in-time recovery Item-level or full share
SQL Server in VM RSV Transaction log chain, crash-consistent Database-aware restore
Managed Disk Backup Vault Cloud-first, snapshot-based, cheaper Disk snapshot restore
Azure Blob Backup Vault Continuous operational backup, PITR Block blob version restore
AKS Cluster Backup Vault Etcd + PV backup, cluster-level restore Cluster-level restore
App Service Backup Vault Code + config backup (preview) Application artifact restore

Key Differentiators

💰 Cost Optimization

Backup Vault is cheaper for short-retention cloud-native workloads. Managed disk snapshots cost less than RSV monthly charges.

Example: 500GB disk backup: Backup Vault ~$15/month vs RSV ~$50/month

⚡ Simplicity

Backup Vault is purpose-built for cloud-native. Simpler UI, fewer configuration options, SLA-driven retention.

Result: Faster onboarding, fewer operational decisions

🔄 Flexibility

Recovery Services Vault supports 1-9999 day retention. Backup Vault enforces SLA-based retention (1-35 days).

Tradeoff: Less flexibility, but better compliance alignment

🎯 Future-Proof

Backup Vault is Microsoft's strategic direction for cloud backup. New features and workloads prioritize Backup Vault.

Strategy: Adopt Backup Vault early for greenfield projects

Demo Scenarios

Scenario 1: Vault Comparison & Selection (5 minutes)

Objective: Show vault features and help select appropriate vault for workload

# 1. Show Recovery Services Vault features az backup vault show -g protection-rg -n rsv-backup-modern # 2. Show Backup Vault features az dataprotection backup-vault show -g protection-rg -n bkpvault-backup-modern # 3. Present workload placement decision matrix echo "Use RSV for: VMs, Files, SQL Server, On-Premises Use Backup Vault for: Managed Disks, Blob, AKS, App Services"

Talking Points:

  • "RSV is mature and flexible, Backup Vault is cloud-native and simpler"
  • "Disk snapshots are cheaper than RSV monthly charges"
  • "Choose based on workload type, not personal preference"
Scenario 2: Managed Disk Backup & Restore (10 minutes)

Objective: Show snapshot-based disk backup and restore

# 1. Enable backup for managed disk az dataprotection backup-instance create \ --datasource-id /subscriptions/.../disks/managed-disk \ --backup-vault-name bkpvault-backup-modern \ -g protection-rg # 2. Trigger immediate snapshot az dataprotection backup-instance adhoc-backup \ --backup-instance-name disk-backup \ --backup-vault-name bkpvault-backup-modern # 3. List created snapshots az snapshot list -g protection-rg \ --query "[].{name:name, size:diskSizeGb}" # 4. Restore from snapshot (create disk) az disk create --name restored-disk \ -g workload-rg --source /subscriptions/.../snapshots/backup-snapshot

Talking Points:

  • "Snapshots are instant and immutable"
  • "No monthly vault charge for disk backups"
  • "Restore is faster than RSV disk recovery"
Scenario 3: Blob Operational Backup & PITR (8 minutes)

Objective: Show continuous point-in-time restore (PITR)

# 1. Enable operational backup for blobs az dataprotection backup-policy create \ --policy-name blob-operational \ --datasource-type AzureBlob \ --backup-vault-name bkpvault-backup-modern # 2. Upload files az storage blob upload -c backup-demo \ -n "file-$(date +%s).txt" --account-name storage # 3. Delete a file (accidental deletion) az storage blob delete -c backup-demo \ -n "file-*.txt" --account-name storage # 4. Restore to point-in-time (before deletion) az dataprotection backup-instance restore trigger \ --restore-to-timestamp "2026-06-23T14:30:00Z" # 5. Verify file restored az storage blob list -c backup-demo --account-name storage

Talking Points:

  • "Blob backup is continuous, no job scheduling needed"
  • "7-day PITR window (shorter, cheaper than RSV)"
  • "Perfect for data lakes and unstructured data"

Frequently Asked Questions

Why are there two vault types?

RSV was built for traditional IaaS workloads with flexible retention. Backup Vault is purpose-built for cloud-native with SLA-driven retention and lower cost.

Can I use Backup Vault for VMs?

Backup Vault doesn't currently support VM backup. Use RSV for VMs, Backup Vault for disks/blobs/AKS/app services.

What's the restore time difference?

RSV file restore: 3-5 minutes, Disk restore: 5-15 minutes, Backup Vault disk snapshot restore: < 1 minute, Blob PITR: < 1 minute

Is Backup Vault production-ready?

Yes, Backup Vault is generally available (GA) for Managed Disks, Blob, and AKS. App Service backup is in preview.

How do I migrate from RSV to Backup Vault?

For managed disks, create new backup policies in Backup Vault. For blob storage, enable operational backup and disable RSV backup.
