Pattern 1 — Centralized Hub AI
Overview

A single shared Azure OpenAI instance behind API Management serves the entire organization. Azure AI Foundry provides a unified orchestration layer for prompt flows, evaluations, and model management. APIM acts as a GenAI gateway with token rate-limiting, semantic caching, and circuit-breaker policies.

Key Services
  • Azure OpenAI Service (gpt-4o, gpt-4o-mini, o1-mini, text-embedding-3-large)
  • Azure AI Foundry — unified AI development & orchestration
  • API Management (Standard v2) — GenAI gateway with semantic caching
  • Azure AI Search — vector search + semantic ranker (hybrid retrieval)
  • Azure AI Content Safety
  • Key Vault, Azure Monitor, Log Analytics, Microsoft Entra ID
When to Use
  • Org-wide GenAI rollout with unified policy & token governance
  • Cost-conscious deployments with shared capacity pooling
  • Teams that don't need model customization or fine-tuning

APIM Standard v2 (GenAI gateway)$700

Azure AI Foundry (free workspace tier)$0
Key Vault + Monitor + Content Safety~$40
Total~$3,400/mo

Pattern 2 — Decentralized Spoke AI
Overview

Each business unit owns its own Azure OpenAI, AI Foundry workspace, and networking resources within a dedicated spoke VNet. Azure Virtual Network Manager provides centralized governance across all spoke topologies.

Key Services
  • Per-team Azure OpenAI (gpt-4o, gpt-4o-mini)
  • Azure AI Foundry workspace per spoke
  • Azure Virtual Network Manager — centralized network governance
  • Spoke VNets with peering, Azure Firewall, NSGs
  • Dedicated Key Vaults & AI Search (vector) per spoke
When to Use
  • Teams needing full autonomy over AI resources & models
  • Different compliance / data residency requirements per BU
  • Large enterprises with independent P&Ls and separate AI budgets

Pattern 3 — Hybrid Central-Spoke
Overview

Central APIM GenAI gateway + shared AOAI with spoke-level fine-tuned models. AI Foundry hub project provides shared prompt flows, evaluations, and model catalog; spoke projects enable team-level experimentation. APIM load-balances across multiple AOAI endpoints with semantic caching and circuit-breaker policies.

Key Services
  • APIM as GenAI gateway (token rate-limiting, semantic caching, load balancing)
  • Azure AI Foundry — hub project + spoke projects
  • Hub AOAI (gpt-4o, gpt-4o-mini, o1-mini) + spoke ML endpoints
  • Azure AI Search S1 (vector + semantic ranker)
  • Prompt flow for AI orchestration
  • Hub-spoke VNet topology, Azure Firewall
When to Use
  • Enterprise needing both shared & custom models
  • Centralized policy with spoke-level experimentation
  • Organizations adopting AI Foundry for unified development

AI Evaluation scores model quality, safety, and groundedness. Supports both Azure OpenAI and open-source models from the AI Model Catalog (Llama 3, Mistral, Phi-3).

Key Services
  • Azure AI Foundry — unified AI platform (replaces standalone Azure ML)
  • Prompt flow — orchestration of LLM chains, RAG, agents

  • Azure AI Agent Service — autonomous AI agents
  • Container Registry, Key Vault, Azure DevOps / GitHub Actions
  • GPU compute: NC A100 v4 (1× A100 80GB)
When to Use
  • Organizations training, fine-tuning & deploying custom models
  • Teams needing reproducible AI/ML pipelines with CI/CD
  • Regulated environments requiring model lineage & responsible AI

Pattern 5 — Serverless AI
Overview

Event-driven AI processing with Azure Functions Flex Consumption (faster cold starts, VNet integration, instance concurrency), Azure Container Apps for containerized AI microservices, and consumption-based pricing throughout. Durable Functions orchestrate multi-step AI workflows (RAG chains, document processing).

Key Services
  • Azure Functions (Flex Consumption) — faster cold starts, VNet support
  • Azure Container Apps — serverless containers with scale-to-zero
  • Durable Functions — multi-step AI orchestration
  • Azure OpenAI (gpt-4o-mini pay-per-token, o1-mini for reasoning)
  • Logic Apps, Event Grid, Cosmos DB serverless
  • AI Search Basic (vector + semantic ranker)
When to Use
  • Bursty or unpredictable AI workloads
  • Startups / cost-sensitive deployments needing near-zero idle cost
  • Document processing, chatbots, event-driven AI pipelines

Pattern 6 — Secure Private AI
Overview

Zero-trust AI deployment with private endpoints, confidential VMs (DCesv5 / ECesv5), CMK encryption, and full network isolation. Azure AI Foundry configured with managed VNet isolation. Microsoft Entra Workload ID replaces stored service-principal secrets. Optional Azure Confidential Ledger for tamper-proof audit trails.

Key Services
  • Azure AI Foundry — managed VNet isolation mode
  • Azure OpenAI (gpt-4o, o1-mini) via Private Endpoints only
  • Confidential VMs (DCesv5 — AMD SEV-SNP)
  • Azure Firewall Premium (IDPS + TLS inspection)
  • DDoS Network Protection (renamed from Standard)
  • Managed HSM, Customer-Managed Keys (double encryption)
  • Microsoft Entra Workload ID — secretless identity
  • Azure Confidential Ledger — immutable audit trail
When to Use
  • Healthcare, finance, government & defense workloads
  • Data that must never traverse public internet
  • Compliance: HIPAA, SOC 2, FedRAMP, PCI-DSS, NIST 800-53

  • Azure Update Manager — unified patching (replaces Update Management)

Azure Verified Modules (AVM) ensure production-grade IaC. Microsoft Entra ID Governance manages lifecycle & access reviews. Azure Monitor managed Grafana provides unified dashboards.

  • Azure Verified Modules (AVM) for IaC
  • Management Groups + Azure Policy + Blueprints
  • Azure Firewall (hub egress) + VPN Gateway / ExpressRoute
  • Azure OpenAI + AI Foundry + AI Search
  • Microsoft Entra ID Governance — access reviews, PIM, lifecycle
  • Defender for Cloud CSPM + Azure Monitor managed Grafana
  • Microsoft Copilot for Azure — AI-assisted operations
When to Use
  • No existing Azure footprint — starting fresh
  • Cloud-first startup or enterprise digital transformation
  • Regulated industry needing governance from day one
  • Building an AI-ready platform from scratch
