Deployment Patterns

Choose the right architecture based on your governance, isolation, cost, and compliance needs.

Pattern 1
Centralized AI Platform
Shared AI platform in the Hub, consumed by multiple spokes via VNet peering.
Low Cost Medium Complexity Platform LZ
  • Azure OpenAI
  • AML
  • AI Search
  • ADLS Gen2
  • Key Vault
  • VNet Peering
Use Cases Enterprise copilots · Shared AI services · Controlled cost & governance
Pattern 2
Decentralized AI per Workload
Each application owns its complete AI stack. Full isolation, no hub dependency.
High Cost Medium Complexity App LZ
  • Azure OpenAI
  • AI Search
  • ADLS Gen2
  • AKS
  • Key Vault
Use Cases Data isolation (PII/PCI/PHI) · Independent teams · High throughput
Pattern 3
Hybrid Central + Spoke
Base models in Hub, inference & RAG in spokes. Best of both worlds.
Medium Cost Medium-High Complexity Platform + App LZ
  • Azure OpenAI (Hub)
  • AML (Hub)
  • AI Search (Spoke)
  • App Service / AKS
Use Cases CSI vertical apps · RAG applications · Controlled model reuse
Pattern 4
AI Factory (MLOps)
Dedicated AI Factory LZ for the full ML lifecycle — training to serving.
High Cost High Complexity AI Factory LZ
  • AML Pipelines
  • Databricks
  • ADLS Gen2
  • ACR
  • Azure OpenAI
Use Cases Large-scale ML pipelines · Continuous retraining · Enterprise MLOps
Pattern 5
Serverless AI
Minimal infrastructure, fully managed. Deploy an AI stack in minutes.
Low Cost Low Complexity Sandbox LZ
  • Azure OpenAI
  • Functions
  • AI Search
  • Blob Storage
  • APIM (opt)
Use Cases Internal copilots · MVPs / PoC · Rapid prototyping · Hackathons
Pattern 6
Secure Private AI (Zero Trust)
Fully private network boundary. No public access. Firewall-controlled egress.
Very High Cost Very High Complexity Platform LZ
  • Azure Firewall
  • Private AKS
  • OpenAI (Private)
  • AI Search (Private)
  • All PEs
Use Cases Data residency (Canada) · Financial / Healthcare · Regulated customers
Pattern 7
Data-Centric Lakehouse AI
AI tightly integrated with a Lakehouse. Medallion architecture + vector search.
High Cost High Complexity Data + App LZ
  • Databricks
  • ADLS Gen2 ×3
  • AI Search
  • Azure OpenAI
  • App Service
Use Cases RAG on enterprise data · Knowledge copilots · Analytics + AI workloads

Architecture Previews

High-level topology for each pattern.

1. Centralized Hub AI
Hub VNet (10.0.0.0/16) ├── Azure OpenAI (PE) ├── AI Search (PE) ├── AML + ADLS (PE) └── Key Vault (PE) │ VNet Peering ┌─────┴──────┐ │ Spoke 1 │ Spoke 2 ... │ AKS/App │ AKS/App └────────────┘
2. Decentralized Spoke AI
Workload VNet (per app) ├── Azure OpenAI (PE) ├── AI Search (PE) ├── ADLS Gen2 (PE) ├── Key Vault (PE) └── AKS Cluster ├── system pool └── gpu pool (opt)
3. Hybrid Central + Spoke
Hub VNet ├── OpenAI (base) (PE) └── AML Registry (PE) │ VNet Peering Spoke VNet ├── AI Search (RAG) (PE) ├── ADLS (docs) └── App Service / AKS
4. AI Factory (MLOps)
Factory VNet ├── Databricks (VNet-injected) │ ├── public /22 │ └── private /22 ├── AML Workspace (PE) ├── ADLS Features (PE) ├── ADLS Models (PE) ├── ACR (PE) └── Azure OpenAI (PE)
5. Serverless AI
Fully Managed (No VNet) ├── Azure OpenAI ├── Azure Functions (Y1) ├── AI Search (Basic) ├── Blob Storage ├── App Insights └── APIM (optional)
6. Secure Private AI
Secure VNet (10.30.0.0/16) ├── Azure Firewall (Premium) │ └── UDR 0.0.0.0/0 → FW ├── Private AKS (NSG) ├── OpenAI (PE, no pub) ├── AI Search (PE, no pub) ├── ADLS + KV (PE, no pub) └── AML (PE, no pub) publicNetworkAccess = Disabled
7. Data-Centric Lakehouse
Lakehouse VNet ├── Databricks (VNet-injected) ├── ADLS Raw → landing/raw ├── ADLS Curated → silver/gold ├── ADLS Serving → embeddings ├── AI Search (vector layer) ├── Azure OpenAI (inference) └── App Service (RAG app)

Deploy Any Pattern

Each pattern is independently deployable with a single Azure CLI command.

# Choose your pattern (1-7) az deployment group create \ --resource-group rg-ai-pattern-dev \ --template-file patterns/<N>-<pattern>/main.bicep \ --parameters patterns/<N>-<pattern>/main.bicepparam # Example: Deploy Serverless AI (Pattern 5) az deployment group create \ --resource-group rg-serverless-ai-dev \ --template-file patterns/5-serverless-ai/main.bicep \ --parameters patterns/5-serverless-ai/main.bicepparam
