Restore Workflows (Demo Scenarios)
Scenario 1: File-Level Restore (Fastest)

Use Case: "I accidentally deleted important Excel files"

Time: 3-5 minutes

Steps:
  1. Open Recovery Services Vault → demo-vault-prod
  2. Select "Protected Items" → demo-vm-prod
  3. Click "Recover Files" (from recovery point)
  4. Choose backup date (e.g., "2 days ago")
  5. Select files to restore: budget.xlsx, reports.pdf
  6. Download restore .iso file
  7. Mount .iso on VM, copy files to C:\
$ az backup recoverypoint list --vault-name demo-vault-prod --container-name demo-vm-prod
[
{
"name": "recoverypoint-2026-06-23",
"timestamp": "2026-06-23T23:00:00Z",
"size_in_gb": 45,
"status": "Healthy"
}
]
$ File restore initiated - .iso ready for download

Talking Point: "File restore is what 80% of customers use. It's fast, granular, and requires no full VM recovery."

Scenario 2: Full Disk Restore (Complete Recovery)

Use Case: "Data disk corruption, need to restore entire disk"

Time: 5-10 minutes

Steps:
  1. Open Recovery Services Vault → demo-vault-prod
  2. Select "Protected Items" → demo-vm-prod (data disk)
  3. Click "Restore Disk"
  4. Choose target resource group (demo-test-rg)
  5. Select recovery point (most recent)
  6. Wait 5 minutes for managed disk creation
  7. Attach managed disk to test VM for verification
$ az backup restore restore-disks --vault-name demo-vault-prod --restore-mode OriginalStorageAccount
Restore in progress...

$ Disk ready for attachment to test VM

Talking Point: "Full disk restore lets you test recovery in a separate environment before attaching to production. This is the safest approach for critical workloads."

Cost Optimization Tips
  • Stop VM after demo — Pause compute costs while testing (~$30 saved per day)
  • Delete after demo — Don't leave resources running overnight
  • Use B-series for demos — Burstable VM size is 60% cheaper than D-series
  • 7-day retention is enough — Don't increase retention (costs scale linearly)
  • LRS storage only — Demo doesn't need GRS (geo-redundancy costs 30% more)

Frequently Asked Questions
How long does the demo take?

Pattern A typically takes 30-45 minutes from start (showing vault) to finish (restore complete). Total setup (including data seeding) takes 60 minutes.

Can I run this demo without login?

Yes! This demo page is pre-authenticated with the demo-user account. No login required — all functionality is available immediately.

What if the backup job hasn't completed yet?

You can use the sample output shown in this demo page. The actual demo scripts include fallback sample data if backup is still in progress.

How much will this cost my company?

Pattern A costs ~$47/day while all resources are running. You should delete resources immediately after the demo (don't leave running overnight). Total cost is typically $47-95 per demo.

Can I customize the demo data?

Yes! The seed-demo-data.ps1 script is fully customizable. You can add customer-specific files, different sizes, or business-relevant documents.

What if I need a Linux VM instead of Windows?

The Bicep template supports both. Just change the parameter deployWindowsVm=false to deploy Ubuntu 22.04 instead. Backup workflows are identical.

Can I extend retention beyond 7 days?

Yes, change the retentionDays parameter in the Bicep template. Note: retention costs scale linearly. 30 days = 4x cost, 90 days = 12x cost.

What's the difference between soft delete and immutable vault lock?

Soft delete: Allows you to undelete accidentally deleted backups for 14 days. Immutable vault lock: Prevents vault deletion permanently (useful for compliance). Pattern A uses soft delete; Pattern B adds immutable lock.

How do I escalate to customer questions about RBAC or enterprise governance?

Use Pattern B for ransomware resilience / RBAC questions, or Pattern D for enterprise governance / cost allocation questions. Each pattern targets specific stakeholders.

Can I run multiple demos concurrently?

Yes, each demo uses a unique resource group (rg-demo-backup-core-{random}). You can run multiple instances in parallel in the same subscription.
