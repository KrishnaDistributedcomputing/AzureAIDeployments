Working Model

Start with the architecture outcome, not the resource list. A module spec should explain why the service exists, how it fits the landing zone, and what downstream teams can rely on.
  1. Choose the target module or pattern and capture the intended business capability.
  2. Define interfaces: identity, networking, secrets, telemetry, private access, and outputs.
  3. Apply the shared ruleset so teams inherit consistent design constraints.
  4. Map the approved contract into Bicep and supporting documentation.
