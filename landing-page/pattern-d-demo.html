Pattern D: Enterprise Landing Zone

CAF-Aligned Governance, Zero-Trust Security & FinOps Cost Control

Enterprise Production Pattern

Pattern Overview

Pattern D demonstrates enterprise-grade Azure Backup governance aligned with Microsoft Cloud Adoption Framework (CAF). Covers three-tier resource separation, zero-trust networking, policy-driven compliance, FinOps cost controls, and operational runbooks.

🎯 Target Audience Enterprise architects, CIOs, governance officers, CAF practitioners, operations teams
⏱️ Deployment Time 40-50 minutes automated deployment

Three-Tier Resource Group Separation:

Workload RG (App Owners)

Owner: Application teams

Contents: Production VMs, databases, application storage

Backup Policy: Daily, retention by compliance level

Protection RG (Backup/Security)

Owner: Backup and security teams

Contents: Recovery Services Vault, Backup Vault, private endpoints, NSGs

Access: Limited to backup operators and security staff

Operations RG (SOC/FinOps)

Owner: Security Operations and FinOps teams

Contents: Log Analytics workspace, storage account (audit logs), Azure Bastion, Azure Monitor

Access: Limited to compliance and finance teams

Role-Based Access Control (RBAC)

Pattern D implements 5 roles with clear separation of duties across backup, security, and finance teams:

Backup Administrator

Full vault management, policy creation, retention decisions

2-3 people, high trust

Backup Operator

Trigger backups, view recovery points, no policy changes

App teams, limited permissions

Restore Operator

Restore workflows only, no backup configuration

Incident response teams

Vault Reader

Read-only access to vault status, logs, policies

SOC, audit teams, CIO office

Cost Analyst

Query Log Analytics for cost tracking and budget alerts

FinOps team

Separation of Duties: Each team can perform their role without interfering with other teams' responsibilities. This prevents unauthorized changes and supports compliance audits.

Tagging Strategy (CAF-Aligned)

Mandatory tags enforced via Azure Policy enable cost allocation, compliance tracking, and automated operations:

Tag Purpose Example Values
environment Controls retention and redundancy production, staging, development
cost-center Chargeback and budget tracking IT-001, IT-002, Finance-001
backup-policy Auto-enrollment to backup policies daily-7, daily-30, weekly-90
compliance-level Retention tier: 7/30/90 days INTERNAL, CONFIDENTIAL, RESTRICTED
owner Team responsible, alert routing database-team@company.com

Example VM Tags (RESTRICTED):

environment: production cost-center: IT-001 backup-policy: daily-30 compliance-level: RESTRICTED owner: database-team@company.com

Zero-Trust Network Security

Pattern D eliminates internet-exposed backup infrastructure using private endpoints and Azure Bastion:

🔒 Private Endpoints

Vaults accessible only via private network. No public IP, no internet exposure.

Benefit: DDoS protected, compliance-aligned, ransomware can't reach vault

🖥️ Azure Bastion

SSH/RDP access to VMs without public IPs or jump boxes.

Benefit: Secure admin access, audit logging, no credential exposure

Result: Backup infrastructure has zero internet exposure while maintaining full operational access.

Demo Scenarios

Scenario 1: CAF Three-Tier Governance (8 minutes)

Objective: Show separation of duties and tag-driven policies

# 1. Show three resource groups with different owners az group list -q "[?tags.owner == 'workload-team' || tags.owner == 'security-team' || tags.owner == 'finops-team']" # 2. Show mandatory tags enforced az group show -n rg-backup-enterprise-workload --query tags # 3. Show RBAC role assignments (5 distinct roles) az role assignment list --scope /subscriptions/$(az account show -q id -o tsv) # 4. Show Azure Policy definitions (tag enforcement) az policy definition list --query "[?contains(displayName, 'require-tag')]"

Talking Points:

  • "Three teams own their resource groups: no single point of failure"
  • "Tags drive backup policies automatically"
  • "Azure Policy prevents tag deletion, ensuring compliance"
Scenario 2: Zero-Trust Network & Bastion (6 minutes)

Objective: Show private endpoints and Bastion access

# 1. Show private endpoints (vaults not public) az network private-endpoint list -g rg-backup-enterprise-protection # 2. Show Azure Bastion host az network bastion list -g rg-backup-enterprise-operations # 3. Demonstrate Bastion RDP to VM (no RDP on public internet) # In Azure Portal: Bastion → Connect → VM Name → RDP # 4. Verify NSGs block unapproved traffic az network nsg list -g rg-backup-enterprise-protection

Talking Points:

  • "Vault has no public IP address"
  • "All traffic flows through private network"
  • "Bastion provides secure admin access without exposing VMs"
Scenario 3: FinOps & Cost Control (8 minutes)

Objective: Show cost tracking and budget alerts

# 1. Run cost-by-cost-center query az monitor log-analytics query \ --workspace law-backup-enterprise \ --analytics-query "@queries/cost-by-cost-center.kql" # 2. Show budget alert rules az monitor metrics alert list -g rg-backup-enterprise-operations # 3. Estimate monthly costs by compliance level # Query: Show daily costs × 30 days # 4. Show chargeback allocation echo "INTERNAL (7d): $20/month per 100GB CONFIDENTIAL (30d): $70/month per 100GB RESTRICTED (90d): $180/month per 100GB"

Talking Points:

  • "Different compliance levels have different retention costs"
  • "Chargeback by cost center prevents central IT from absorbing all costs"
  • "Budget alerts prevent runaway spending"
Scenario 4: Compliance Audit Trail (6 minutes)

Objective: Show immutable audit logs and compliance proof

# 1. Show compliance audit query (RESTRICTED workloads) az monitor log-analytics query \ --workspace law-backup-enterprise \ --analytics-query "@queries/compliance-audit-trail.kql" # 2. Show immutable vault lock (permanent protection) az backup vault get -g rg-backup-enterprise-protection \ -n rsv-backup-enterprise-cc | jq '.properties.immutabilitySettings.state' # 3. Show storage account with immutable blobs (audit logs) az storage account show \ -g rg-backup-enterprise-operations \ -n storageaudit... | jq '.properties.immutableStorageWithVersioning' # 4. Show soft delete enabled (14-day recovery) az backup vault backup-properties show \ -g rg-backup-enterprise-protection \ -n rsv-backup-enterprise-cc | jq '.softDeleteFeatureState'

Talking Points:

  • "Immutable lock proves backups can't be deleted (compliance proof)"
  • "Audit logs are immutable for regulatory audits (HIPAA, PCI-DSS, SOX)"
  • "Soft delete provides 14-day ransomware recovery window"

Frequently Asked Questions

Why three resource groups?

Separation of duties. Different teams manage different layers. If one team's account is compromised, attackers can't access other RGs.

Do I need all 5 RBAC roles?

No. Start with Backup Admin, Operator, and Restore Operator. Add Vault Reader and Cost Analyst as your organization grows.

Can I use Backup Vault instead of RSV?

Partially. Use Backup Vault for managed disks, blobs, and AKS. Use RSV for VMs and file shares. Both are deployed in Pattern D.

How much does this cost?

Depends on workload size. Estimate: $60-80/day for the infrastructure (2 vaults, 3 RGs, private endpoints, Bastion) + data backup costs.

Is this CAF compliant?

Yes. Pattern D implements CAF governance model: resource group separation, tagging strategy, RBAC separation, policy-driven compliance.
